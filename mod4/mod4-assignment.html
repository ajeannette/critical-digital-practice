For this activity you're going to make a webpage on GitHub. 

Flat-file

We're going to keep it "old school" here by creating a flat-file website, which, unlike fully configured platforms like WordPress, are simply a collection of individual text files stored somewhere on your server, rather than in a database. Flat-file websites are fantastic for:

  • Simplicity
  • Speed
  • Security
  • Version Control
  • Portability and maintenance
  • Less expense to run

(What is a Flat-File CMS?)

Content Management Systems (CMSs) are great for this, and a particularly popular one that we use here at the CDL, called Jekyll, is made for GitHub pages. These CMSs, essentially, provide powerful templates for your content and files, relieving you from a lot of work. However, we won't be using a flat-file CMS for this activity either. 

No, rather, you're going to build all of the pages you need for this website from scratch. What you'll end up with will certainly not be as fancy as most modern websites, and, in fact, will likely be pretty reminiscent of older site styling such as MySpace and other early-aught webpages. But, much like the creators of those sites, you'll learn a ton. Don't worry, it'll be fun!

The website

Your website can be on any topic, but there are a few things that you should include. 

You site should have more than one page, be navigable, have a header, footer, and a style.

The steps

To complete this activity, you'll need to do the following:

  1. Create a GitHub repository and name it yourusername.github.io.
  2. Clone your repository to your computer.
  3. Create an index.html file in your repository folder.
  4. Create a style.css file in your repository folder and link to it in your index.html file.
  5. Create!

When you're finished with this activity, post the link to your GitHub repository in the Teams chat.
